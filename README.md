# Olist 巴西电商平台数据分析

## 一、项目背景

Olist 是巴西最大的电商 SaaS 平台。本项目基于 Olist 公开的真实交易数据，
遵循"大盘诊断 → 归因分析 → 策略落地"的闭环分析思路，从核心指标、时间趋势、
地域分析、品类分析、订单履约五个维度进行基础诊断，并在此基础上对用户分层、
物流效率、支付行为、商品与卖家生态四个方向做深入归因，最终输出可落地的业务建议。

## 二、数据集说明

- **数据来源**：Kaggle 公开数据集 Brazilian E-Commerce Public Dataset by Olist
- **数据规模**：约 10 万笔订单，覆盖 2016–2018 年，共 9 张数据表
- **脱敏情况**：数据已做隐私脱敏，企业名称替换为虚拟名称
- **核心表结构**：

| 数据表 | 说明 |
|--------|------|
| orders | 订单主表（订单状态、各环节时间戳） |
| order_items | 订单商品明细（价格、运费） |
| order_payments | 支付信息（支付方式、分期数、金额） |
| order_reviews | 订单评价（评分、评论） |
| customers | 客户信息（地理位置） |
| sellers | 卖家信息（地理位置） |
| products | 商品信息（品类、尺寸、重量） |
| geolocation | 地理坐标信息 |
| product_category_translation | 品类葡萄牙语—英语对照表 |

各表通过 order_id、customer_id、seller_id、product_id 等主外键关联。

## 三、技术栈

- **SQL**：多表关联与聚合分析，完成数据提取
- **Pandas / Python**：数据清洗与处理
- **Matplotlib / Seaborn**：可视化分析
- **Tableau**：交互式商业看板

## 四、分析流程
```
数据提取(SQL) → 数据清洗(Pandas) → 五个维度的基础分析 → 四个方面进行深入探究→ 商业看板(Tableau) → 业务结论与建议
```

## 五、业务分析

### 5.1 平台基础诊断

1. 平台总订单量与独立访客数十分接近，说明**复购率极低、回头客稀少**
2. 交易额与订单数持续递增，客单价稳定在 **120–130 R$** 的较高区间
3. GMV 高度集中，**SP 州贡献近半交易额**，省会周边几个州贡献主要 GMV
4. **超过 60% 的交易为跨州交易**
5. 美妆、手表礼品等 **17 个核心品类贡献约 80% 销售额**
6. 平台整体履约率达 **97%**，运行状态总体健康

### 5.2 核心问题归因

**问题一：客户流失严重，营收结构脆弱**
- 通过 RFM 用户分层，超过 50% 为"重要挽留"与"重要发展"客户，另有约 25% 已流失
- 上述两类客户贡献了 80% 以上营收，平台缺乏稳定的忠实客户基本盘

**问题二：物流质量低下**
- 受巴西地域辽阔 + 60% 跨州交易影响，各州平均配送天数普遍偏高，
  AP、AM 等偏远州的平均配送天数逼近 30 天
- 多数州超期率在 10% 以上，AL 州超期率超过 20%
- 商家处理时长普遍在 2–4 天，是超期的重要诱因之一

**问题三：分期支付带来资金风险**
- 分期付款在巴西电商极为普遍，约 2/3 的客户依赖分期支付
- 庞大的应收账款占用大量资金，叠加当地经济环境，存在潜在坏账风险

**问题四：主力产品分散，竞争力承压**
- 主力品类横跨手表礼品、美妆、电脑配件等多个领域，缺乏拳头产品
- 易受垂直电商或拥有招牌品类的综合平台冲击

**问题五：卖家生态失衡**
- 符合典型二八法则，**前 10% 卖家贡献近 70% GMV**
- 头部卖家平均评分却低于中小卖家，反映平台对支柱卖家监管不足

## 六、业务建议

1. **深度绑定用户**：当前增长更像红利驱动的"虚假繁荣"，应加大基础设施与
   用户运营投入，以短期成本换取用户长期留存
2. **建设本地仓储**：物流困境根源在于地理，单纯增效收效有限，
   建议布局区域前置仓，将货物集中到离消费者更近的区域处理
3. **完善资金管理**：针对分期账单建立科学的应收管理与风控体系，
   提升资金周转、防范坏账
4. **聚焦垂直深耕**：在综合平台定位下，针对性深耕优势品类，增强消费者信任
5. **强化头部治理**：头部卖家是平台支柱，需加强约束并协助其建立更优服务体系
